- Key decisions: The weekday automation runs hourly from 10:00 through 20:00 only; manual refresh remains available. Feishu records are read-only in the dashboard and now open their Feishu source instead of exposing a non-working completion control. Feishu Tasks expose only open/completed, deadlines, and timestamps, not “in progress” or “review”; overview cards show open, overdue, due-this-week, and completed counts. Assignments that are both overdue and inactive for 14 days are omitted from active views because Feishu retains historical assignments after their owner leaves. Project progress is calculated from workflow completion across synchronized Project work items. Suggested ToDos from meeting notes and messages require both an explicit action cue and either a Jessie mention or a configured Real-time/EOD business keyword; the keyword lists include bilingual market-data, portfolio, technical-analysis, risk, report, view, mobile, and widget terms in `sync.config.json`. The dashboard displays a 96-character title preview and a navigable local Codex session ID for ranked token use; sessions with missing model metadata are labelled as legacy. GitHub API failures show the available local repository history, which is paged to keep the panel compact. The page-level language choice is stored locally; the news view switches content language and the control is available globally.
